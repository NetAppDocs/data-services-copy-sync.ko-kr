= NetApp Copy and Sync를 위한 빠른 시작
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ./media/


[role="lead"]
NetApp Copy and Sync를 시작하려면 몇 가지 단계가 필요합니다.

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-1.png["하나"]로그인하고 NetApp 콘솔을 설정하세요
[role="quick-margin-para"]
NetApp 콘솔을 시작했어야 합니다. 여기에는 로그인, 계정 설정, 콘솔 에이전트 배포 및 시스템 생성이 포함됩니다.

[role="quick-margin-para"]
다음 중 하나에 대한 동기화 관계를 만들려면 먼저 시스템을 만들거나 검색해야 합니다.

[role="quick-margin-list"]
* ONTAP 용 Amazon FSx
* Azure NetApp Files
* Cloud Volumes ONTAP
* 온프레미스 ONTAP 클러스터


[role="quick-margin-para"]
Cloud Volumes ONTAP, 온프레미스 ONTAP 클러스터 및 Amazon FSx for ONTAP 에는 콘솔 에이전트가 필요합니다.

[role="quick-margin-list"]
* https://docs.netapp.com/us-en/bluexp-setup-admin/concept-overview.html["NetApp 콘솔을 시작하는 방법을 알아보세요"^]
* https://docs.netapp.com/us-en/bluexp-setup-admin/concept-connectors.html["콘솔 에이전트에 대해 자세히 알아보세요"^]


.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-2.png["둘"]소스와 타겟을 준비하세요
[role="quick-margin-para"]
소스와 타겟이 지원되고 설정되어 있는지 확인하세요.  가장 중요한 요구 사항은 데이터 브로커 그룹과 소스 및 타겟 위치 간의 연결을 확인하는 것입니다.

[role="quick-margin-list"]
* link:reference-supported-relationships.html["지원되는 관계 보기"]
* link:reference-requirements.html["소스와 타겟을 준비하세요"]


.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-3.png["삼"]NetApp 데이터 브로커를 위한 위치 준비
[role="quick-margin-para"]
NetApp 데이터 브로커 소프트웨어는 소스에서 타겟으로 데이터를 동기화합니다(이를 _동기화 관계_라고 합니다).  AWS, Azure, Google Cloud Platform 또는 사내에서 데이터 브로커를 실행할 수 있습니다.  하나 이상의 데이터 브로커로 구성된 데이터 브로커 그룹은 NetApp Copy and Sync와 통신하고 몇몇 다른 서비스와 저장소에 접속할 수 있도록 포트 443을 통한 아웃바운드 인터넷 연결이 필요합니다. link:reference-networking.html#networking-endpoints["엔드포인트 목록 보기"] .

[role="quick-margin-para"]
NetApp Copy and Sync는 동기화 관계를 생성할 때 설치 과정을 안내하며, 이때 클라우드에 데이터 브로커를 배포하거나 자체 Linux 호스트에 대한 설치 스크립트를 다운로드할 수 있습니다.

[role="quick-margin-list"]
* link:task-installing-aws.html["AWS 설치 검토"]
* link:task-installing-azure.html["Azure 설치 검토"]
* link:task-installing-gcp.html["Google Cloud 설치 검토"]
* link:task-installing-linux.html["Linux 호스트 설치 검토"]


.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-4.png["네"]첫 번째 동기화 관계를 만드세요
[role="quick-margin-para"]
로그인하세요 https://console.netapp.com/["NetApp 콘솔"^] , *동기화*를 선택한 다음 소스와 대상에 대한 선택 항목을 끌어서 놓습니다.  화면의 지시에 따라 설정을 완료하세요. link:task-creating-relationships.html["자세히 알아보기"] .

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-5.png["다섯"]무료 평가판이 종료된 후 동기화 관계에 대한 비용을 지불하세요.
[role="quick-margin-para"]
AWS 또는 Azure에 가입하여 사용량에 따라 요금을 지불하거나 연간 요금을 지불하세요.  또는 NetApp 에서 직접 라이선스를 구매하세요.  NetApp Copy and Sync의 라이선스 설정 페이지로 가서 설정하기만 하면 됩니다. link:task-licensing.html["자세히 알아보기"] .
